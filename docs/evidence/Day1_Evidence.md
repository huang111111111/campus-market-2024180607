Task 1
# 一、为什么要开发校园轻集市
1. **校园内二手、闲置交易需求旺盛，但缺少专属线上渠道**
学生有大量闲置教材、宿舍用品、运动器材、数码配件、服饰、考研资料等，线下摆摊受时间、场地、天气限制，微信群、QQ群交易杂乱，信息刷屏易丢失，无统一发布、检索渠道。
2. **现有通用二手平台不适配校园场景**
闲鱼、转转等面向全社会，距离远、运费高、存在校外人员交易风险，缺少校园认证、自提、校园专属分类，学生使用体验差。
3. **校园资源流转效率低，资源浪费严重**
毕业生大量闲置物品直接丢弃，低年级学生重复购买新品，造成物资浪费；同时学生有短期租借、物品互换、拼单团购需求，无对应平台承载。
4. **实训教学目标：贴合学生真实场景练手**
选用校园集市作为实训项目，业务逻辑简单易懂、贴近学生日常生活，便于快速理解需求，同时完整覆盖前后端、登录、发布、检索、聊天、收藏等Web开发核心知识点，适合七天短期实训。
5. **校园安全管理需求**
平台可做本校学生身份校验，限制校外人员入驻，规避线下私下交易带来的财物诈骗、人身安全隐患，规范校内物品交易行为。

# 二、系统主要解决哪些实际问题
1. **信息分散杂乱问题**
替代零散微信群、朋友圈、线下摆摊，统一线上发布入口，商品分类清晰，支持关键词搜索、筛选，闲置信息不会被刷屏淹没。
2. **交易距离与成本问题**
仅限本校学生交易，支持宿舍/教学楼线下自提，省去快递运费，降低交易成本，交易见面地点可控在校内公共区域。
3. **身份与安全风险问题**
依托校园账号/学号完成身份核验，杜绝校外人员混入交易；商品发布可审核，减少虚假物品、诈骗、违规商品（违规电器、违禁品）。
4. **资源浪费与重复消费问题**
搭建循环流转渠道，教材、家具、小家电等闲置物品二次流通，减少学生重复购买，降低学生生活开支。
5. **多元需求无承载渠道问题**
兼顾二手转卖、物品租借、好物互换、校园拼单、求购信息多种场景，统一在一个系统实现，不用切换多个社交软件。
6. **交易无留存、无保障问题**
聊天记录、交易记录平台留存，支持收藏心仪商品、发布求购帖，方便学生沟通溯源，对比线下口头交易更有凭证。
7. **实训层面：解决学生缺少完整项目实战的问题**
提供一套完整可落地的Web业务系统，让学生完整走完需求、架构、编码、联调、部署全流程，掌握企业级开发流程。

# 三、系统包含哪些业务场景
## 1. 用户身份场景
- 学生注册、登录、退出登录；
- 校园身份校验（学号/账号认证）；
- 个人中心：修改头像、昵称、查看我的发布、我的收藏、我的聊天、交易记录。

## 2. 商品发布与管理场景
- 发布闲置商品：上传图片、填写名称、价格、分类、描述、新旧程度；
- 编辑、下架、删除自己发布的商品；
- 商品后台简易审核（过滤违规内容）。

## 3. 商品浏览检索场景
- 首页轮播推荐热门商品；
- 分类导航：教材、数码、宿舍用品、服饰、运动器材、租借专区等；
- 关键词搜索、价格筛选、按发布时间排序；
- 商品详情页查看大图、发布者信息、收藏商品。

## 4. 互动交易场景
- 求购专区：学生发布求购帖子，等待卖家联系；
- 私聊聊天：买家与卖家在线沟通价格、自提时间；
- 收藏、点赞心仪商品；
- 线下自提交易（线上沟通，线下当面交付，平台不做线上支付）。

## 5. 特色校园业务场景
- 物品短期租借（充电宝、相机、球拍、书籍租赁）；
- 好物互换（以物换物，不涉及金钱）；
- 校园拼单、团购发布。

## 6. 管理员后台场景
- 用户管理、商品审核、违规商品下架；
- 分类管理、首页推荐配置；
- 数据简单统计：发布量、活跃用户。

# 四、七天实训最终要完成什么
## 1. 文档交付成果
1. 完整《需求分析文档》：梳理用户角色、业务场景、功能清单、原型说明；
2. 《技术架构文档》：前后端技术栈、项目目录结构、数据库表设计、接口规范；
3. 《实训方案+实训计划》整理归档，梳理开发分工、每日任务节点；

## 2. 代码开发成品
1. 前端页面完整实现：首页、商品分类、搜索、商品详情、发布页、个人中心、聊天页、登录注册、求购页；
2. 后端接口全部联调完成：登录、商品增删改查、收藏、聊天、用户信息接口；
3. 数据库设计并导入测试数据，可正常存储用户、商品、聊天记录；
4. 实现双语切换、页面响应式布局、导航栏、弹窗、轮播等交互效果；
5. 解决登录鉴权、图片上传、筛选检索、人机交互等核心功能bug，系统稳定可运行。

## 3. 运行与演示成果
1. 项目本地完整部署，执行`npm install`、`npm run dev`可一键启动，无依赖报错；
2. 完整演示流程：注册登录→发布商品→浏览搜索商品→私聊沟通→收藏/发布求购；
3. 代码规范统一，无冗余报错，文件分层清晰，注释完整。

## 4. 实训汇报成果
1. 制作PPT：讲解项目背景、需求、架构、核心功能、开发难点；
2. 完整演示系统全部业务流程；
3. 提交实训总结，梳理开发过程遇到的问题、解决方案、实训收获。

## 5. 整体目标
独立完成一套**校园轻集市完整可运行Web应用**，从需求分析、架构设计到编码、联调、演示全流程落地，掌握Vue3、Bootstrap、前后端交互、数据库设计等实训核心技术，达到小型实战项目开发能力。
